What is Git?

Git is a distributed version control system: tracking changes in any set of files, usually used for coordinating work among programmers collaboratively developing source code during software development. Its goals include speed, data integrity, and support for distributed, non-linear workflows

Git commands demo

Here we'll create a simple git repository and manage it via git (windows), using different commands and also look how to manage different issues regarding git, like merging, branching, etc.

1. init command

The git init command is used to create an empty Git repository or reinitialize an existing one. It is used to start a new repository from scratch, or to reinitialize an existing one. It creates a new subdirectory named .git that contains all of your necessary repository files — a Git repository skeleton. This subdirectory is what is copied when you clone a repository from another computer.

2. add command

The git add command adds a change in the working directory to the staging area. It tells Git that you want to include updates to a particular file in the next commit. It adds the file to the staging area, but it doesn’t actually commit it to the repository yet. You can think of the staging area as a prep table where Git will take the next commit.

3. commit command

The git commit command is used to save your changes to the local repository. It takes a snapshot of the project's currently staged changes. This command saves the changes to the local repository with a message that describes the changes.

commit command

4. push command

The git push command is used to upload local repository content to a remote repository. Pushing is how you transfer commits from your local repository to a remote repo. It's the counterpart to git fetch, but whereas fetching imports commits to local branches, pushing exports commits to remote branches.

5. clone command

The git clone command is used to create a local copy of a remote repository. Cloning a repository creates a new directory, initializes it with a .git directory, pulls down all the data for that repository, and checks out a working copy of the latest version. If you want to clone a repository into a directory named something other than repo, you can specify the new directory name as an additional argument.

6. pull command

The git pull command is used to fetch and download content from a remote repository and immediately update the local repository to match that content. Pulling is how you transfer commits from a remote repository to your local repo. It's the counterpart to git push, but whereas pushing exports commits to remote branches, pulling imports commits to local branches.

Here, If changes were made by someone else on the remote repository, we can pull those changes to our local repository. If changes were made and we try to push them, we'll get an error, 'failed to push some refs to remote'. So, we'll pull the changes first and then push them.

7. branch command

The git branch command is used to list, create, or delete branches. A branch is an independent line of development. It is contained within the repository, but does not affect the primary or master branch allowing you to work freely without disrupting the "live" version. When you want to add a new feature or fix a bug—no matter how big or how small—you spawn a new branch to encapsulate your changes. This makes sure that unstable code is never committed to the main code base, and it gives you the chance to clean up your future's history before merging it into the main branch.
